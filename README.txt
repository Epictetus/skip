* はじめに
================================================================================

1. SKIP(Social Knowledge & Innovation Platform)とは
--------------------------------------------------------------------------------
社内コミュニケーションを豊かにするための、
  企業内に散在する"暗黙知"を"見える化"して繋げていく、
    ナレッジマネジメントを実現する社内SNSです。

2. 公式サイト
--------------------------------------------------------------------------------
http://www.openskip.org/

3. お問い合わせ
--------------------------------------------------------------------------------

--------------------------------------------------------------------------------

================================================================================
* SKIP 動作環境
================================================================================
SKIPを動かすにあたり以下の環境で検証を行っています。
（動作を保証するものではありません、また、下記環境以外でも動作する可能性はあります）

 - OS    : Linux (kernel 2.6系)
 - DB    : MySQL 5.0系
 - Ruby  : 1.8.6, 1.8.7
 - Rails : 2.1.1


================================================================================
* SKIP セットアップ
================================================================================
下記内容に従いセットアップすることで、SKIPを動かすことができます。

1. Ruby, RubyGems, MySQL等、各種ソフトウェアのインストール
--------------------------------------------------------------------------------
 1. yumを使ってインストール（RedHat or fedoraユーザの場合）

    $ sudo yum install ruby rubygems ruby-devel mysql mysql-server ruby-mysql zlib-devel

2. gemsのインストール
--------------------------------------------------------------------------------
 1. 順番にインストール

    $ sudo gem install gettext
    $ sudo gem install uuidtools
    $ sudo gem install fastercsv
    $ sudo gem install ruby-openid
    $ sudo gem install rails --version 2.1.1

3. SKIPの展開
--------------------------------------------------------------------------------
 1. 公式ページから最新版SKIPをダウンロード

    -> http://www.openskip.org/download/

 2. 解凍する

    $ tar xvzf skip-1.0rc2.tar.gz

 3. 設定ファイルの作成

    $ cd [RAILS_ROOT]/config
    $ cp initial_settings.yml.sample initial_settings.yml
    $ cp database.yml.sample database.yml
    $ cd ../

 4. config/initial_settings.ymlを編集

    session:のsecretを変更する(30文字以上にする必要があります)
    sha1_digest_keyを変更する
    secret_keyを変更する
    hostを起動するURLに変更する
    proxy_urlにプロキシサーバのIPアドレスを設定する(必要な場合のみ)
    その他、必要に応じて変更する

4. データベースの作成
--------------------------------------------------------------------------------
 1. mysqlを起動

 2. mysqlのデータベースを作成

    $ rake db:create RAILS_ENV="production"

 3. rakeコマンド実行

    $ rake db:migrate RAILS_ENV="production"

5. 初期管理者ユーザ登録URLの生成
--------------------------------------------------------------------------------

    $ ruby lib/create_new_admin_url.rb --code

6. サーバ起動
--------------------------------------------------------------------------------
 1. サーバを起動する

    $ ruby ./script/server -p [initial_settingsで指定したSKIP起動ポート] -e production

 2. 初期管理者ユーザを作成する

    6. で表示されたURLにブラウザでアクセスして管理者ユーザを登録する。


================================================================================
* SKIPのユーザ追加
================================================================================
SKIPを起動した直後はユーザが存在していません。
管理画面から、ユーザの追加を行う必要があります。
管理画面には、セットアップ時に作成したユーザでログインするとアクセスできます。
（右側メニューの一番下）

1. 管理者でログインする
--------------------------------------------------------------------------------
 - 7.2で作成した初期管理者ユーザでログインしてください。

2. 利用可能ユーザの登録
--------------------------------------------------------------------------------
>>一人ずつ登録する場合
 - システム管理 -> データ管理 -> ユーザ -> 新規ユーザの作成 から ユーザを作成する

>>CSVで一括登録する場合
 - システム管理 -> データ管理 -> ユーザ -> ユーザのCSVアップロード から アップロードする

3. IDとパスワードを、ユーザに連絡する
--------------------------------------------------------------------------------
 - IDとパスワードをユーザに連絡する
    (初期パスワードは必ず変更するようにユーザに伝達すること)
