基本情報
================================================================================

1. SKIP(Social Knowledge & Innovation Platform)とは
--------------------------------------------------------------------------------
社内コミュニケーションを豊かにするための、企業内に散在する"暗黙知"を"見える化"して繋げていく、ナレッジマネジメントを実現するSNSです。

2. 公式サイト
--------------------------------------------------------------------------------
http://www.openskip.org

3. ライセンス
--------------------------------------------------------------------------------

4. クレジット
--------------------------------------------------------------------------------
icons by famfamfam.com (http://www.famfamfam.com/lab/icons/silk/)

================================================================================
SKIP インストール方法
================================================================================

for Unix (Linux, etc.)
================================================================================

1. Ruby, RubyGems, MySQL等、各種ソフトウェアのインストール
--------------------------------------------------------------------------------
Debian 及び Ubuntuユーザの場合
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
 1. yumを使ってインストール

    $ sudo yum install ruby rubygems ruby-devel mysql mysql-server ruby-mysql zlib-devel

2. gemsのインストール
--------------------------------------------------------------------------------
 1. 順番にインストール

    $ sudo gem install gettext
    $ sudo gem install uuidtools
    $ sudo gem install fastercsv
    $ sudo gem install ruby-openid
    $ sudo gem install rails --version 2.1.0

3. SKIPの展開
--------------------------------------------------------------------------------
 1. 公式ページから最新版SKIPをダウンロード。(http://www.openskip.org/download/)
 2. 解凍する

    $ tar xvzf skip-1.0rc1.tar.gz

 3. 設定ファイルの作成

    $ cd [RAILS_ROOT]/config
    $ cp initial_settings.yml.sample initial_settings.yml
    $ cp database.yml.sample database.yml

 4. initial_settings.ymlを編集
    session:のsecretを変更する(※30文字以上にする必要があります)
    sha1_digest_keyを変更する
    secret_keyを変更する

 5. 依存ファイル(plugin等の展開)

    $ rake skip:setup_production

4. 環境変数の設定
--------------------------------------------------------------------------------
 1. 必要な環境変数を設定する

    $ export SKIP_URL=http://[サーバのIPアドレス or ドメインネーム]:[SKIP起動ポート]
    $ export RAILS_ENV=production

5. データベースの作成
--------------------------------------------------------------------------------
 1. mysqlを起動
 2. mysqlのデータベースを作成 

    $ mysql -u root
    mysql> create database skip_production charset = utf8;

 3. rakeコマンド実行

    $ rake db:migrate

6. プロキシサーバの設定(必要な場合のみ)
--------------------------------------------------------------------------------
 1. initial_settings.ymlを編集

    proxy_urlにプロキシサーバのIPアドレスを設定する
    

7. サーバ起動
--------------------------------------------------------------------------------
 1. サーバを起動する

    $ ruby ./script/server -p [環境変数の設定で、指定したSKIP起動ポート] -e production

================================================================================
SKIP 初期設定
================================================================================

1. initial_settings.ymlの設定
--------------------------------------------------------------------------------
 1. initial_settings.ymlを編集
    必要に応じて変更する

2. 初期管理者ユーザの作成
--------------------------------------------------------------------------------
 1. 初期管理者ユーザ登録URLの生成

    $ export SKIP_HOST=[サーバのIPアドレス or ドメインネーム]:[SKIP起動ポート]
    $ cd [RAILS_ROOT]/lib
    $ ruby create_new_admin_url.rb --code

 2. 初期管理者ユーザを作成する
    
    1. で表示されたURLにブラウザでアクセスして管理者ユーザを登録する。

3. 利用可能ユーザの登録
--------------------------------------------------------------------------------
一人ずつ登録
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
 1. 管理者でログインする
 2. システム管理 -> データ管理 -> ユーザ -> 新規ユーザの作成 から ユーザを作成する

CSVで一括登録
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
 1. 管理者でログインする
 2. システム管理 -> データ管理 -> ユーザ -> ユーザのCSVアップロード から アップロードする

4. IDとパスワードを、ユーザに連絡する
--------------------------------------------------------------------------------
 1. IDとパスワードをユーザに連絡する。
    ※初期パスワードは必ず変更するようにユーザに連絡すること

5. 管理者グループの作成
--------------------------------------------------------------------------------
 1. 管理者でログインする
 2. システム管理 -> その他設定 で 運営チームのグループIDを管理者グループとして使いたいIDに変更する
 3. マイページ -> グループ -> グループの新規作成から上記で作成したグループIDでグループを作成する
